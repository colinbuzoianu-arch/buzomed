## What changes

### Dashboard
After login, you land on `/dashboard` instead of `/team`. The dashboard shows:

**Needs attention** (only visible when there's something to act on):
- Overdue recalls (red card) — links to `/examinations?tab=scadente&horizon=overdue`
- In-progress examinations — links to the in_curs tab
- Unsigned completed examinations — links to the finalizate tab

**Today** (always visible):
- Examinations scheduled for today
- Recalls due this week
- Total exams this month
- Active employee count

**Quick actions**: `+ Examinare nouă`, `+ Angajat nou`, `+ Companie nouă` (based on role), `Scadențe`, `Rapoarte`

**Cabinet overview** at the bottom: company count, employee count, link to full reports.

The greeting ("Bună dimineața", "Bună ziua", "Bună seara") is based on the server's local hour at render time. The user's first name is used. Cabinet name from the Tenant record.

### Toasts
Every major action now gets a bottom-right toast confirmation:
- Save company → "Companie salvată · [name]"
- Delete company → "Companie eliminată · [name]"
- Save employee → "Angajat salvat · [name]"
- Sign fișă → "Fișă semnată · Nr. 2026/0001"
- Cancel/no-show exam → "Examinare anulată"
- Schedule recall → "Programare creată · [worker name]"
- Cancel recall → "Programare anulată"
- Bulk import → "Import finalizat · X angajați importați cu succes"
- Update user account → "Cont actualizat · [name]"
- Archive user → "Cont arhivat · [name]"

### Loading skeletons
The three most-visited pages (Examinations, Companies, Employees) now show an animated skeleton when loading. Next.js uses the `loading.tsx` file in each route segment — the skeleton appears automatically while the server component fetches data.

### PDF fișă
On the fișa de aptitudine page, there's now a **"Descarcă PDF"** button next to "Tipărește". Clicking it downloads `fisa_aptitudine_2026-0001.pdf` — a clean A4 PDF generated by `@react-pdf/renderer`.

What the PDF includes:
- Cabinet name and address in the header
- "FIȘA DE APTITUDINE" title + examination number
- Verdict in a prominent colored box (green for apt, amber for conditioned/temporary, red for unfit)
- Worker details (name, birth date, gender)
- Company / workplace / job title / examination type
- Vital signs (if any were recorded)
- Clinical findings (if present)
- Next examination date
- Practitioner signature block
- "DRAFT" watermark in red if the examination hasn't been signed yet

The PDF uses the Buzomed brand colors (deep blue #1e3a8a, teal #0d9488). The Inter font is loaded from Google Fonts via the React-PDF font registry — if the server is offline, it falls back to Helvetica.

## What @react-pdf/renderer is and isn't

**Is**: a React-tree-to-PDF renderer that uses Yoga flexbox. Runs in Node.js without Chromium. Works on Vercel's free tier. Supports custom fonts.

**Isn't**: a CSS → PDF tool. The layout is defined using Yoga flex properties (think React Native StyleSheet). The PDF fișă is a clean re-implementation of the layout, not a screenshot of the HTML page.

**Limitation**: the PDF doesn't include charts, stamps, or scanned signatures. Signature images are in the schema (`signatureImageUrl`, `stampImageUrl`) but are not yet fetched in the PDF route — that's a future improvement once practitioners configure their profiles.

**Performance**: first render of a PDF takes ~500ms–1s (font loading). Subsequent renders in the same cold start are cached by React-PDF's font registry. Not fast enough for bulk PDF export (future session), but perfectly acceptable for single-document download.

## Why the dashboard defaults to server time for the greeting

The greeting ("Bună dimineața" vs "Bună ziua") uses `new Date().getHours()` on the server. For a Romanian cabinet, the server is in UTC+3 (Romania/EEST in summer, EET in winter). Vercel's servers are in UTC, so the hour will be off by 2-3 hours.

If this is annoying in practice (getting "good evening" at 7pm local time), the fix is to pass the client's timezone as a search param or read it from a cookie. Deferred — the greeting is decorative. Practitioners won't care about a 2-hour greeting offset.

## Commit structure (one bundle, one commit is fine this time)

Session 13 has no schema changes, no permission logic, no security surface. All four features are cosmetic improvements and one new file-download feature. Safe to commit as one:

